## Overview

The Crime Reporting module allows law enforcement personnel to create detailed incident reports (preventivos judiciales) with comprehensive information about crimes, suspects, victims, and evidence.

## Creating a Crime Report

<Steps>
  <Step title="Access the Report Form">
    Navigate to the crime reporting form (`alta_preventivos1.php`) from the main menu.
  </Step>
  
  <Step title="Enter Basic Information">
    Fill in the required preventivo details including number, year, and dependency information.
  </Step>
  
  <Step title="Complete Crime Details">
    Provide comprehensive information about the incident, location, and involved parties.
  </Step>
  
  <Step title="Upload Attachments">
    Add any relevant photos or documents to support the report.
  </Step>
  
  <Step title="Submit the Report">
    Review and submit the preventivo for processing.
  </Step>
</Steps>

## Required Fields

### Report Identification

<ParamField path="nro_preventivo" type="number" required>
  The preventivo number assigned to this incident report
</ParamField>

<ParamField path="anio_preventivo" type="number" required>
  The year of the preventivo (automatically defaults to current year)
</ParamField>

<ParamField path="dependencia" type="select" required>
  The police dependency/station handling the case. Selected from available dependencies in your regional unit.
</ParamField>

### Date and Time Information

<ParamField path="fecha_preventivo" type="date" required>
  Date when the preventivo was created. Must be:
  - Not in a closed period
  - Not in the future
  - Within the last year
</ParamField>

<ParamField path="fecha_hecho" type="date" required>
  Date when the crime actually occurred
</ParamField>

<ParamField path="horario_hecho" type="select" required>
  Time range when the incident occurred. Options include:
  - 00:00 - 02:00
  - 02:00 - 04:00
  - 04:00 - 06:00
  - And so on in 2-hour intervals through 22:00 - 24:00
</ParamField>

### Complainant Information

<ParamField path="denunciante" type="text">
  Name of the person filing the complaint
</ParamField>

<ParamField path="tipoDoc" type="select">
  Type of identification document
</ParamField>

<ParamField path="dni" type="text">
  Identification document number of the complainant
</ParamField>

### Suspect/Author Information

<ParamField path="sospechosos" type="text">
  Names or descriptions of suspects (up to 75 characters)
</ParamField>

<ParamField path="inculpado" type="select" required>
  Whether the suspect is:
  - CONOCIDO (Known)
  - DESCONOCIDO (Unknown)
</ParamField>

<ParamField path="sexo_inculpado" type="select" required>
  Sex of the suspect:
  - MASCULINO
  - FEMENINO
  - NO CONSTA (Not stated)
</ParamField>

<ParamField path="genero_inculpado" type="select" required>
  Gender identity of the suspect (from `identidad_genero` catalog)
</ParamField>

<ParamField path="edad_inculpado" type="select" required>
  Age range of the suspect:
  - MENOR DE 16 (Under 16)
  - DE 16 A 17 AÑOS (16 to 17 years)
  - MAYOR DE 18 AÑOS (Over 18 years)
  - NO CONSTA (Not stated)
</ParamField>

### Victim Information

<ParamField path="sexo_victima" type="select" required>
  Sex of the victim:
  - MASCULINO
  - FEMENINO
  - NO CONSTA
</ParamField>

<ParamField path="genero_victima" type="select" required>
  Gender identity of the victim
</ParamField>

<ParamField path="edad_victima" type="number" required>
  Age of the victim in years
</ParamField>

<ParamField path="vinculo_imputado_victima" type="select" required>
  Relationship between suspect and victim:
  - 1 - CONYUGE/PAREJA (Spouse/Partner)
  - 2 - EX CONYUGE/EX PAREJA (Ex-spouse/Ex-partner)
  - 3 - HIJO/A (Son/Daughter)
  - 4 - HERMANO/A (Sibling)
  - 5 - PADRE/MADRE (Father/Mother)
  - 6 - OTROS VINCULOS (Other relationships)
  - 7 - SOCIO/A (Business partner)
  - 8 - EMPLEADO (Employee)
  - 9 - EMPLEADOR (Employer)
  - 10 - CLIENTE (Client)
  - 11 - OTRAS RELACIONES NO FAMILIARES (Other non-family relations)
  - 12 - SIN RELACION (No relation)
</ParamField>

## Crime Categorization

### Type of Crime

<ParamField path="hecho" type="select" required>
  Select the type of crime from the `tipos_delitos` catalog. Common categories include:
  - Crimes against persons (DEL.C/LAS PERSONAS)
  - Crimes against property (DEL.C/LA PROPIEDAD)
  - Sex crimes (DEL.C/INTEG.SEXUAL)
  - Other crimes (OTROS)
</ParamField>

<ParamField path="modusoperandi" type="select" required>
  Select the modus operandi from the `modus_operandi_preventivos` catalog describing how the crime was committed
</ParamField>

<ParamField path="esclarecido" type="select" required>
  Whether the crime has been solved:
  - SI (Yes)
  - NO (No)
</ParamField>

<ParamField path="motivo_origina_instruccion_causa" type="select" required>
  What initiated the investigation:
  - DENUNCIA PARTICULAR (Private complaint)
  - INTERVENCION POLICIAL (Police intervention)
  - ORDEN JUDICIAL (Court order)
  - OTROS (Other)
</ParamField>

### Recovery of Stolen Items

<ParamField path="recupero_sustraido" type="select">
  Whether stolen items were recovered
</ParamField>

  - ESTABLECIMIENTO SALUD - HOSPITAL (Health facility/Hospital)

</ParamField>

<ParamField path="altura_lugar_hecho" type="text" required>

## Photo/Document Attachments

The system supports uploading one photo per preventivo:

<Steps>
  <Step title="Select File">
    Use the file upload field to select an image file
  </Step>
  
  <Step title="File Naming">
    The file will be automatically renamed to: `{nro_preventivo}-{anio_preventivo}-{dependencia}-{original_filename}`
  </Step>
  
  <Step title="Storage">
    Files are stored in the `../fotos/` directory and referenced in the `fotografias` table
  </Step>
</Steps>

## Validation Rules

<Warning>
  The system enforces several validation rules:
  
  - **Period Closure**: Cannot create preventivos for dates in closed periods
  - **Future Dates**: Cannot create preventivos with future dates
  - **Date Range**: Preventivo date must be within the last year
  - **Duplicate Check**: System checks for duplicate preventivos (same number, year, and dependency)
  - **Required Selections**: All dropdown fields marked as required must be selected
  - **Coordinates**: Map coordinates are mandatory
</Warning>

## Updating Reports

Existing preventivos can be updated by:

1. Searching for the preventivo using its number, year, and dependency
2. The system will load the existing data
3. Modify the necessary fields (primarily coordinates and modus operandi)
4. Submit to update

<Note>
  The update functionality is limited to certain fields. Core incident data cannot be modified after initial creation.
</Note>

## Best Practices

<AccordionGroup>
  <Accordion title="Accurate Location Data">
    Always use the map interface to select precise coordinates. This data is used for spatial analysis and crime mapping.
  </Accordion>
  
  <Accordion title="Complete Victim/Suspect Information">
    Provide as much detail as possible about victims and suspects, including age, sex, and gender identity. This information is crucial for statistical reporting.
  </Accordion>
  
  <Accordion title="Timely Reporting">
    Create preventivos as soon as possible after the incident. Remember that dates cannot be in closed periods or be too far in the past.
  </Accordion>
  
  <Accordion title="Proper Classification">
    Select the most specific crime type and modus operandi available. This ensures accurate crime statistics and trend analysis.
  </Accordion>
  
  <Accordion title="Vehicle Information">
    For vehicle-related crimes, ensure all identification numbers (license plate, chassis, engine) are entered correctly for tracking in the stolen vehicles database.
  </Accordion>
</AccordionGroup>
