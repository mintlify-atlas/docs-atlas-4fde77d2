## Overview

The Seguridad system provides a comprehensive navigation structure that organizes police operations data into logical modules. The main navigation menu adapts based on user roles and provides access to data entry, consultation, and management functions.

## Main Menu Structure

The primary navigation is implemented through a horizontal menu bar (`menu.php`) using the Spry Menu Bar framework. This menu is available to most users after authentication.

<Info>
  The navigation menu is included in most pages through `<?php include ('menu.php');?>` and provides consistent access to system features.
</Info>

## Common Workflows

### Workflow 1: Recording a New Incident

<Steps>
  <Step title="Login to System">
    Authenticate with your credentials at `index.php`
  </Step>
  
  <Step title="Navigate to CARGA DE DATOS">
    Click on **CARGA DE DATOS** in the main menu
  </Step>
  
  <Step title="Select Incident Type">
    Choose the appropriate entry form:
    - **Preventivos** for general cases
    - Specialized forms for specific incident types
  </Step>
  
  <Step title="Complete Entry Form">
    Fill in all required fields:
    - Date and time
    - Location
    - Incident details
    - Involved parties
    - Evidence and observations
  </Step>
  
  <Step title="Submit and Verify">
    Save the record and note the generated report number for future reference
  </Step>
</Steps>

### Workflow 2: Querying Incident Data

<Steps>
  <Step title="Access CONSULTA Module">
    Click **CONSULTA** in the main navigation
  </Step>
  
  <Step title="Select Query Type">
    Choose from:
    - **Delitos** for crime statistics
    - **Mapa** for geographic visualization
    - **Buscar Preventivo** for specific report lookup
  </Step>
  
  <Step title="Set Query Parameters">
    Configure filters:
    - Date range
    - Location/jurisdiction
    - Incident type
    - Additional criteria
  </Step>
  
  <Step title="Execute Query">
    Submit the query and review results
  </Step>
  
  <Step title="Export or Print">
    Generate reports or export data as needed
  </Step>
</Steps>

### Workflow 3: Managing Operations

<Steps>
  <Step title="Plan Operation">
    Navigate to **CARGA DE DATOS** > **Operativos**
  </Step>
  
  <Step title="Select Operation Type">
    - **Operativos** for standard operations
    - **Operativos Integrales** for multi-unit operations
  </Step>
  
  <Step title="Register Operation Details">
    Document:
    - Operation name and objective
    - Participating units
    - Date/time and location
    - Resources allocated
  </Step>
  
  <Step title="Execute and Update">
    Record operation progress and results in real-time
  </Step>
  
  <Step title="Review Results">
    Access **CONSULTA** > **Operativos** for statistical analysis and mapping
  </Step>
</Steps>

### Workflow 4: Shift Chief Reporting

<Steps>
  <Step title="Determine Shift Period">
    - **07-19** for day shift
    - **19-07** for night shift
  </Step>
  
  <Step title="Access Parte Form">
    Navigate to **CARGA DE DATOS** > **PARTE JEFE DE TURNO**
  </Step>
  
  <Step title="Complete Shift Summary">
    Document:
    - Notable incidents
    - Personnel status
    - Resource availability
    - Operational highlights
  </Step>
  
  <Step title="Submit Report">
    Save the shift report for supervisor review
  </Step>
  
  <Step title="Review Historical Reports">
    Access **CONSULTA** > **Parte Jefe de Turno** to view past reports
  </Step>
</Steps>

## Navigation Tips

<Tabs>
  <Tab title="Keyboard Shortcuts">
    While specific keyboard shortcuts aren't documented in the source code, standard browser shortcuts apply:
    
    - **Tab** - Navigate between form fields
    - **Enter** - Submit forms (when in input fields)
    - **Esc** - Close modal dialogs
    - **Ctrl+F** - Search within page
  </Tab>
  
  <Tab title="Browser Compatibility">
    The system is designed for:
    - Modern browsers with JavaScript enabled
    - Spry framework support (Adobe Spry MenuBar)
    - CSS3 and HTML5 compatibility
    
    <Warning>
      For best results, use updated versions of Chrome, Firefox, or Edge.
    </Warning>
  </Tab>
  
  <Tab title="Performance Tips">
    - Use specific date ranges in queries to improve performance
    - Bookmark frequently accessed pages
    - Clear browser cache if experiencing issues
    - Limit map visualizations to relevant data ranges
  </Tab>
</Tabs>

## Module Access by Role

| Menu Section | Admin (0) | Regional Units (1-15) | Judicial (20-28) | Specialized (90-99) |
|--------------|-----------|----------------------|------------------|--------------------|
| **CARGA DE DATOS** | ✓ | ✓ | ✗ | ✓ (limited) |
| **CONSULTA** | ✓ | ✓ | ✓ | ✓ |
| **Operativos Entry** | ✓ | ✓ | ✗ | ✓ (some units) |
| **Usuarios** | ✓ | ✗ | ✗ | ✗ |
| **CONTROL DE CARGA** | ✓ | ✓ | ✗ | ✗ |

<Note>

## Troubleshooting Navigation

<AccordionGroup>
  <Accordion title="Menu Not Displaying">
    **Symptoms:** Navigation menu is blank or not visible
    
    **Solutions:**
    - Verify JavaScript is enabled
    - Check that SpryAssets are loading correctly
    - Clear browser cache and reload
    - Ensure you're authenticated (session valid)
  </Accordion>
  
  <Accordion title="Missing Menu Options">
    **Symptoms:** Some menu items don't appear
    
    **Solutions:**
    - Verify your user role has access to those modules
    - Contact administrator to confirm permissions
    - Check if you're in the correct landing page for your role
  </Accordion>
  
  <Accordion title="Page Not Loading After Menu Click">
    **Symptoms:** Clicking menu items results in errors
    
    **Solutions:**
    - Check session hasn't expired (look for authentication errors)
    - Verify file paths exist on server
    - Review browser console for JavaScript errors
    - Contact system administrator if persistent
  </Accordion>
</AccordionGroup>
